## Goal

依据本仓库的写作规范，生成一篇结构清晰、朴素叙事、有图有出处的中文 AI 技术文章，落盘到 `<article-dir>/article.md`，可直接同步到用户的 `github/book` 仓库。

## Preconditions

- 用户已经提供：主题、目标目录路径（如 `~/Desktop/book/anthropic-skills-spec/`）、源材料（URL / 粘贴文本 / 草稿任一种均可）。
- 缺任意一项时，先问清楚，不要自行设定主题或路径。
- 如果主题明显不属于 AI / ML / Agent / LLM / 工程化等范畴，先和用户确认是否仍要继续，再开始。

## Steps

### 1. 确认输入

向用户确认四件事，缺哪问哪：
- **主题**：一句话能说清楚的题目。
- **目标目录**：绝对路径。目录不存在时创建。
- **源材料**：URL 列表（必须）/ 粘贴的英文原文 / 用户已有的草稿笔记。**没有源材料的纯观点文不写**。
- **长度**：`短文（<800 字）` 或 `深度（1500–2500 字）`。

### 2. 拉取源材料 + 提取事实

- 对每个 URL 用 WebFetch 取正文；记录原作者、发布日期、标题、URL。
- 把里面的**事实声明**和**作者观点**分开记录：事实可以直接引用，观点必须标注 "原文：……"。
- 出现互相矛盾的源时，**两边都引用**并标注分歧，不要替读者选边。

### 3. 加载规范（每次都读）

开始下笔前必须读取以下两个文件作为本次写作的硬约束：

1. `${SKILL_DIR}/references/style-guide.md` —— 10 条风格规则 + 反模式清单。
2. `${SKILL_DIR}/references/citation-conventions.md` —— 引用格式、callout 标签、翻译标注规则。

### 4. 起草纲要

输出一个三段式纲要给用户口头确认（或在 commit message / 终端打印）：
- **开头钩子**：一句话说清"此事此刻为何重要"。禁止"近年来""随着……的发展"这类填充。
- **3–5 个主章节**：每节一个核心论点 + 支撑事实。
- **延伸阅读**：3–5 个高质量外链。

纲要里**每一节都要能回答一句"它解决读者的什么疑问"**——这是写作时的内部自检，**不要写成正文里的 callout 块**。答案要直接体现在小标题和首段里。回答不上来的章节直接删。

### 5. 起草正文

按纲要写正文。每写完一节，对照 `style-guide.md` 自检：

- 每句 ≤ 20 字（中文按字计；硬性上限不是平均）。
- 每段 2–4 句，不超过 200 字。
- 每节 ≤ 1200 字。
- 术语首次出现挂英文原文，例如 "智能体（Agent）"、"提示词缓存（prompt caching）"。
- 任何事实声明都要内联引用 `[原文标题](url)`。
- 任何主观判断都要带 callout 标记（参见 `citation-conventions.md`）。
- 严禁出现 "众所周知""显而易见""毫无疑问""不得不说" 等填充。

### 6. 插入配图占位

在认知负载高的位置（如：解释抽象架构、对比多方案、展示流程）插入图位。**装饰性配图不要**。

每个图位的格式：

```markdown
![FIG-1：三层架构示意图](assets/fig-1.png)
<!-- gen-hint: 一张 9:16 手绘信息图，米色纸面背景，毛笔标题"三层架构"，三块卡片自上而下：用户层 / 编排层 / 模型层，每块卡片配小图标。 -->
```

`gen-hint` 注释里写中文描述即可，下一步 `generate-article-image` skill 会读它生成图片。

### 7. 末尾延伸阅读

每篇文章必须以 `## 延伸阅读` 收尾，列 3–5 个外链。格式：

```markdown
## 延伸阅读

- [原文标题 1](url-1) —— 一句话说清为什么值得读
- [原文标题 2](url-2) —— ……
```

不要塞超过 5 条，不要列没读过的链接。

### 8. 自检 + 落盘

落盘前最后过一遍 `style-guide.md` 的 "反模式" 清单。命中任何一条立刻改：

- [ ] 没有"在本节中我们将……"这类过渡句。
- [ ] 没有不带出处的判断句。
- [ ] 没有连续 >200 字的散文块。
- [ ] 缩写在首次出现时都展开了。
- [ ] 所有图位都带 `gen-hint` 注释。
- [ ] 文末有"延伸阅读"。
- [ ] 没有把"它解决读者什么疑问"这种内部自检写成正文 callout 块。

把最终 markdown 写到 `<article-dir>/article.md`。如果 `<article-dir>/assets/` 不存在，建一个空目录。

### 9. 收尾报告

向用户一句话汇报：
- 落盘路径。
- 字数。
- 图位数量（提示可以下一步用 `generate-article-image` 逐张生成）。
- 引用了哪些原文。

## Success criteria

- `<article-dir>/article.md` 存在。
- 自检清单全过。
- 所有事实声明都有内联出处。
- 所有图位都带 `gen-hint`。
- 文末有"延伸阅读"。

## Stop conditions

- **缺源材料**：用户给的主题没有任何外部出处时，停下问，**不要靠记忆编事实**。
- **源之间冲突**：两个源说法矛盾且都言之有据，停下让用户决定怎么呈现，**不要替用户选**。
- **主题越界**：主题明显不属于 AI 技术写作（例如菜谱、生活随笔），停下确认再继续。
- **目标目录已有 `article.md`**：停下问要覆盖还是改名，**不要静默覆盖**。

## Notes

- 这个 skill 不负责把文章推到 `github/book`。落盘到本地后用户自己同步。
- 想让配图和正文一气呵成，可以在第 6 步把图位先全写好，等用户审稿通过后再批量调 `generate-article-image`。
- 长文（>2500 字）建议拆系列。单篇过长违反 style-guide.md 第 10 条。
